# 03 · 따라해보기 (A) — 코딩 에이전트로 (클로드 코드·Codex 등)

> **두 갈래 중 (A)** 입니다. 본인에게 맞는 길을 고르세요:
> - **(A)** 클로드 코드·Codex·Cursor 같은 **코딩 에이전트**가 있다 → **이 문서**. 자연어로 부탁하면 에이전트가 파일을 만들고 실행까지 합니다.
> - **(B)** 그런 도구는 없고 **AI Studio(브라우저 속 Gemini)** 와 복사·붙여넣기로 손수 → [`03b-tutorial-ai-studio.md`](./03b-tutorial-ai-studio.md)

이제 직접 만듭니다. **자동화 1(사진 → 노션 기록 → 배포)** 을 처음부터 끝까지 따라갑니다.

## 이 문서를 읽는 법 (중요)

각 단계는 **항상 같은 4박자**로 갑니다:

- 🎯 **지금 하는 일** — 무엇을, 왜 (개념은 [01](./01-concepts.md)으로 링크)
- 💬 **클로드에게 이렇게** — 여러분이 한국어로 입력할 **요청문(프롬프트)**
- 📁 **그러면 이런 게 만들어집니다** — 클로드가 짠 결과의 *구조* (코드 말고 "무엇이 생겼나")
- ✅ **체크포인트** — 다음으로 넘어가기 전 "되는지" 확인 / 🆘 **막히면**

> 🙆 **여러분은 코드를 안 짭니다. 안 읽어도 됩니다.** 클로드가 짭니다. 여러분은 *요청하고 → 확인*만 합니다.
> 🤔 클로드가 중간에 되물을 수 있어요("이 표 이름이 맞나요?"). 편하게 한국어로 답하면 됩니다.
> 🎲 클로드가 짠 코드의 모양은 매번 조금씩 다를 수 있습니다. **결과(노션에 기록되는가)만 같으면 정답**입니다. 정답지가 궁금하면 이 폴더의 상위 `lib/`·`api/`를 열어보세요.

## STEP 0 · 준비 (15분)

코딩 전에 멍석을 깝니다.

### 0-1. 계정·키 준비
[가이드 첫 문서의 체크리스트](./README.md#시작-전-준비물-체크리스트)대로 계정 4개를 만들고, **AI 키 하나**를 미리 발급해 두세요:
- **Gemini(이미지 AI) 키**: <https://aistudio.google.com/apikey> → "Create API key" → 나온 문자열을 메모장에 잠깐 보관 (곧 안전한 곳에 옮깁니다)

### 0-2. 노션 표 2개 만들기
노션에서 표(데이터베이스) 2개를 직접 만드세요. 칸 구성은 [02 워크플로의 데이터 모델 표](./02-workflow.md#노션-데이터-모델--어떤-표를-미리-만들어-두나)를 그대로 보고 만들면 됩니다.
- **챌린저 표**: 참가자 몇 명을 넣어두세요 (없으면 화면이 비어 보입니다).
- **러닝 기록 표**: 빈 채로 두면 됩니다. 자동으로 채워질 곳입니다.

> 📌 표를 손으로 만드는 게 더 빠르고 눈에 보입니다. ("클로드가 API로 표도 만들 수 있나요?" → 됩니다. 하지만 입문 단계에선 노션에서 직접 만드는 걸 권합니다.)

### 0-3. 노션 인테그레이션(사원증) 만들고 표에 초대
- <https://www.notion.so/my-integrations> → "New integration" 생성 → **토큰**(`ntn_...`) 복사해 메모장에 잠깐 보관
- 방금 만든 표 2개 각각 우상단 `⋯ → 연결 추가`에서 그 인테그레이션을 선택 → **표에 들어올 권한 부여** (이걸 빼먹으면 나중에 "권한 없음" 오류)

### 0-4. 작업 폴더 만들고 클로드 코드 열기
- 빈 폴더를 하나 만듭니다 (예: `running-challenge`).
- 그 폴더에서 **클로드 코드**를 엽니다.

지금 폴더 상태:
```
running-challenge/
   (아직 비어 있음)
```

✅ **체크포인트**: 노션에 표 2개 + 챌린저 몇 명, 손에 토큰(`ntn_...`)과 Gemini 키, 그리고 빈 폴더에서 열린 클로드 코드.

## STEP 1 · 노션과 연결되는지 확인

🎯 **지금 하는 일**: 우리 코드가 노션에 들어갈 수 있는지부터 확인합니다. (개념: [API](./01-concepts.md#api--프로그램끼리-주고받는-전용-창구), [토큰/사원증](./01-concepts.md#인테그레이션--토큰--출입-사원증과-비밀번호), [시크릿 관리](./01-concepts.md#시크릿-관리--비밀번호는-따로-보관))

💬 **클로드에게 이렇게**:
> 내 노션 '러닝 기록' 표에 연결해서 어떤 칸(속성)들이 있는지 읽어와줘.
> 노션 토큰은 코드에 직접 쓰지 말고 `.env.local` 파일에 넣어서 안전하게 관리해줘.
> 토큰 값은 내가 직접 그 파일에 넣을게 — 파일 틀만 만들어줘.

(그다음 클로드가 알려주는 대로 `.env.local`에 토큰을 붙여 넣습니다. **이 파일은 깃허브에 안 올라갑니다.**)

📁 **그러면 이런 게 만들어집니다**:
```
running-challenge/
  .env.local        ← 토큰 보관 (비밀, 깃허브 제외)
  .gitignore        ← ".env.local은 올리지 마" 규칙
  (노션 호출하는 코드 한 조각)
```
클로드가 노션을 호출해 **칸 목록**(제목, 일시, 거리(km), 챌린저…)을 터미널에 출력해줍니다.

✅ **체크포인트**: 표의 칸 이름들이 주르륵 출력되면 연결 성공.
🆘 **막히면**:
- `401`/권한 오류 → STEP 0-3의 "연결 추가"를 안 했을 가능성. 표에 인테그레이션을 다시 초대.
- `Not found` → 표를 잘못 지정. "내 '러닝 기록' 표 주소(URL) 줄게, 이걸로 연결해줘"라고 URL을 주세요.

## STEP 2 · 사진을 읽는 "두뇌" 만들기 (OCR)

🎯 **지금 하는 일**: 러닝 앱 스크린샷에서 숫자를 뽑아내는 부분을 만듭니다. (개념: [비전 OCR & 구조화 출력](./01-concepts.md#비전-ocr--구조화-출력--사진을-읽고-정해진-칸에-맞춰-답하기))

먼저 테스트용 사진을 준비하세요: 본인 러닝 앱 스크린샷 몇 장을 폴더 안 `samples/`에 넣습니다. (이 repo의 `samples/` 사진을 가져다 써도 됩니다.)

💬 **클로드에게 이렇게**:
> 러닝 앱 스크린샷(애플워치·가민·삼성헬스·NRC 등)에서 거리·시간·페이스·심박수·칼로리를 뽑아내는 코드를 만들어줘.
> Gemini를 쓰고, 키는 `.env.local`의 `GEMINI_API_KEY`로 읽어줘.
> 중요한 규칙: **화면에 안 보이는 값은 추측하지 말고 비워둬.** 거리가 마일이면 km로 바꿔줘.
> 결과는 프로그램이 바로 쓸 수 있게 **정해진 칸으로만(JSON 형식)** 돌려줘.

📁 **그러면 이런 게 만들어집니다**:
```
running-challenge/
  lib/gemini.js     ← 사진 → 데이터. 여기에 "AI에게 줄 지시문 + 답 형식"이 들어 있음
  samples/          ← 테스트용 스크린샷
```

✅ **체크포인트**: "samples 사진 한 장으로 시험해줘"라고 하면, 거리·시간·페이스가 **숫자로** 출력됩니다. 화면에 없던 항목은 비어 있어야 정상.
🆘 **막히면**:
- 안 보이는 값을 그럴듯하게 지어냄 → "추측 금지, 없으면 비워둬"를 프롬프트에 더 강하게.
- 모델 오류 → "최신 Gemini 모델로 바꿔줘"라고 요청.

## STEP 3 · 노션에 넣는 "손" 만들기 + 진짜 되는지 테스트

🎯 **지금 하는 일**: 뽑아낸 데이터를 노션 표에 새 데이터베이스 페이지(표의 한 줄)로 집어넣습니다. (개념: [API로 쓰기](./01-concepts.md#api--프로그램끼리-주고받는-전용-창구), 관계(relation)는 [02 워크플로](./02-workflow.md#노션-데이터-모델--어떤-표를-미리-만들어-두나) 참고)

💬 **클로드에게 이렇게**:
> STEP 2에서 뽑은 데이터를 내 노션 '러닝 기록' 표에 **새 데이터베이스 페이지**로 넣어줘.
> 올린 사진은 '인증샷' 칸에 첨부하고, '챌린저' 칸으로 누가 올렸는지 연결해줘.
> 빈 값(비어 있는 항목)은 칸에 넣지 말고 건너뛰어줘.
> 그리고 `samples/` 사진 한 장으로 **터미널에서 한 번 진짜 돌려봐줘.**

📁 **그러면 이런 게 만들어집니다**:
```
running-challenge/
  lib/notion.js      ← 노션과 대화: 새 페이지 만들기 + 사진 첨부
  cli-upload.mjs     ← 웹 만들기 전, 터미널에서 사진 한 장 테스트하는 도구
```

✅ **체크포인트**: 노션 '러닝 기록' 표를 열어보면 **새 데이터베이스 페이지**가 생기고, 거리·시간 등이 채워지고, 인증샷 사진이 붙어 있고, 챌린저가 연결돼 있습니다. → **여기까지면 자동화의 핵심은 이미 완성**입니다. 남은 건 "사람이 쓰기 편하게 웹으로 감싸기".
🆘 **막히면**:
- 페이지는 생겼는데 챌린저 연결이 안 됨 → "'러닝 기록'의 챌린저 칸이 '챌린저' 표와 연결(relation)돼 있는지" 노션에서 확인.
- 권한/401 → `.env.local` 토큰 확인.

## STEP 4 · 웹페이지로 감싸기 (플랜모드 첫 경험)

🎯 **지금 하는 일**: 챌린저들은 터미널을 못 씁니다. **사진만 올리면 되는 웹페이지 한 장**으로 감쌉니다. (개념: [서버리스 함수](./01-concepts.md#서버리스-함수--필요할-때만-깨어나는-작은-일꾼))

> 💡 **클로드 코드 기능 하나 소개 — 플랜모드.** 큰 작업은 클로드가 코드부터 짜지 않고 **계획을 먼저 세우게** 할 수 있습니다. `Shift+Tab`을 두 번 눌러 "Plan mode"로 바꾸면, 클로드가 "이렇게 만들 건데 괜찮나요?"를 먼저 보여주고 여러분 승인 후 만듭니다. **진짜 중요한 일은 코딩이 아니라 '무엇을 만들지 결정'** 이라서, 이 습관이 큰 도움이 됩니다.

💬 **클로드에게 이렇게** (플랜모드로 전환 후):
> 챌린저가 **사진만 올리면 되는 웹페이지 한 장**을 만들어줘.
> 위쪽엔 '누가 올리는지' 고르는 목록 — 이건 노션 '챌린저' 표에서 가져와줘.
> 사진을 올리면 STEP 2~3 흐름(읽기→노션 기록)이 돌아가고, "이렇게 기록됐어요" 결과를 보여줘.
> 먼저 **계획부터** 보여주고, 내가 OK 하면 만들어줘.

📁 **그러면 이런 게 만들어집니다**:
```
running-challenge/
  public/index.html       ← 챌린저들이 보는 화면 (고르기 + 올리기 + 결과)
  api/challengers.js       ← "참가자 목록 알려줘" 입구
  api/upload.js            ← "사진 받아 처리해줘" 입구
  package.json             ← 이 프로젝트가 쓰는 부품 목록
```

✅ **체크포인트**: "로컬에서 실행해줘"라고 하면 클로드가 미리보기 주소(예: `localhost:3000`)를 띄웁니다. 거기서 본인 고르고 사진 올리면 → 노션에 새 데이터베이스 페이지가 들어갑니다. **내 컴퓨터에서 웹앱이 도는 상태.**
🆘 **막히면**:
- 목록이 비어 있음 → 노션 '챌린저' 표에 사람이 있는지 + 그 표에도 인테그레이션을 초대했는지 확인.
- 키 관련 오류 → `.env.local`에 `NOTION_TOKEN`, `GEMINI_API_KEY`가 다 있는지 확인. (클로드에게 "키 등록 상태 확인하는 점검용 입구 만들어줘"라고 하면 `api/debug.js`를 만들어 줍니다 — 값은 안 보이고 "있다/없다"만 알려줌)

## STEP 5 · 깃허브에 올리기

🎯 **지금 하는 일**: 코드를 인터넷 창고(깃허브)에 백업합니다. 곧 Vercel이 여기서 받아갑니다. (개념: [배포·Vercel의 "창고/출판사" 비유](./01-concepts.md#배포--vercel--내-노트북이-꺼져도-돌아가게))

💬 **클로드에게 이렇게**:
> 이 코드를 깃허브에 **비공개 리포로** 올려줘.

(외울 명령어 없습니다. 클로드가 시크릿이 안 올라가는지 점검 → 리포 생성 → 업로드까지 알아서 합니다.)

📁 **그러면 이런 게 만들어집니다**: 인터넷에 내 코드 저장소(리포) 주소 하나. **단, `.env.local`(비밀)은 올라가지 않습니다.**

✅ **체크포인트**: 깃허브 리포를 열어 **`.env.local`이 목록에 없는지** 눈으로 확인하세요. ([01의 시크릿 관리](./01-concepts.md#시크릿-관리--비밀번호는-따로-보관)에서 말한, `.gitignore`가 막아주는 그 장면입니다.)
🆘 **막히면**: "깃허브 로그인부터 도와줘"라고 하면 로그인 과정을 안내해 줍니다.

## STEP 6 · Vercel에 배포 — 진짜 인터넷 주소 받기

🎯 **지금 하는 일**: 내 노트북이 꺼져도 도는, **누구나 접속할 주소**를 받습니다. (개념: [배포·Vercel](./01-concepts.md#배포--vercel--내-노트북이-꺼져도-돌아가게))

💬 **클로드에게 이렇게**:
> 이걸 Vercel에 배포해줘.
> `.env.local`에 있는 키들을 **Vercel에도 안전하게 등록**해줘. (깃허브엔 안 올라갔으니 Vercel은 따로 알아야 함)

📁 **그러면 이런 게 만들어집니다**: `https://러닝챌린지-어쩌고.vercel.app` 같은 **공개 주소**. 이걸 챌린저들에게 보내면 끝.

✅ **체크포인트**: 그 주소를 **휴대폰 브라우저**로 열어보세요. 본인 고르고, 러닝 앱 스크린샷 한 장 올리면 → 30초 안에 노션에 새 데이터베이스 페이지가 채워집니다. **🎉 자동화 완성.**
🆘 **막히면**:
- 사이트는 열리는데 업로드 시 권한 오류(401) → Vercel에 키가 일부만 등록됐을 수 있음. "Vercel에 등록된 키 목록 확인하고 빠진 거 채워줘"라고 요청.
- 화면은 뜨는데 목록이 빔 → STEP 4의 막히면과 동일(노션 챌린저 표 + 인테그레이션 초대).

## 🎉 끝 — 여러분이 만든 것

```
[챌린저]  주소 접속 → 본인 선택 → 스크린샷 업로드
            → AI가 읽고 → 노션에 자동 기록
```

운영자가 숫자를 옮겨 적던 일이 사라졌습니다. **내 노트북을 꺼도 계속 돌아갑니다.**

### 고친 뒤 다시 올리기 (보너스)
이제 문구나 화면을 바꾸고 싶으면 클로드에게 "○○ 고쳐줘" → "깃허브에 올려줘"라고만 하면, Vercel이 **1분 안에 자동으로 다시 배포**합니다. (코드 고침 = 라이브 즉시 반영)

## 다음 단계 — 심화: 자동 문자 보내기 (선택)

여기까지가 **자동화 1(입력)** 입니다. 더 나아가고 싶다면 **자동화 2(알림)** 가 있습니다:
- 기록이 올라오는 *순간* "오늘도 달리셨네요!" 문자 (= [웹훅](./01-concepts.md#심화-웹훅-vs-크론--자동-문자를-보내는-두-가지-방아쇠))
- 매일 아침, 며칠 쉰 사람에게 독려 문자 (= [크론](./01-concepts.md#심화-웹훅-vs-크론--자동-문자를-보내는-두-가지-방아쇠))

구조와 안전장치(중복 방지·발송 on/off 스위치 등)는 [02 워크플로의 "심화" 절](./02-workflow.md#심화--문자-자동화의-안전장치들)에 정리돼 있습니다. 실제 코드는 이 repo의 `api/webhook/`, `api/cron/`, `lib/solapi.js`에 있으니, 같은 방식으로 클로드에게 한 단계씩 부탁하면 됩니다. (단, 문자 발송은 유료 문자 서비스 계정과 발신번호 등록이 필요합니다.)
